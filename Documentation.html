iForm - form creator & validator

For jQuery & Zurb Foundation framework

Documentation

In this documentation you will find all the information to know how to use the iForm pluguin developed for leganux.


About

The iForm plugin was developed by Angel Erick Cruz Olivera in JS using the ES5 and ES6 standar, the end of the develop of this project was on July 2018, the actual version is 1.0.0.2.

iForm allows you create adaptativee an functional forms for web applications that uses fundation as framework, the forms you will create in it also includes the most important validations to make easiest the way you control all the validations coming from a JSON configuration. This Plugin help you as developer to write less code and permit that the final user sees all the application estandarized. To get the best UX as result.

Enjoy it!
Erick Cruz


Dependences

iForm need a third party elements to work fine. Yo must to include them in your page before start using it.

  • jQuery 3.0.0.+
  • Last version of production ambient of foundation framework



Getting Started

After you shure you include the dependences os this project you must to include the JS library in your project before body tag closes.

<script src="iForm.js?v1.0.0.4"></script>

Now you must to create a div element in the body of your html file and asign to it an unique id because in this place the form will be located.

<div id="MyForm1"></div>

Now in your JS app code you must to inicialice a new instance of the iForm Object, you'll need 2 JSON objects and the id selector of the div you create before.

Wait don't worry... In the next topics of this documentation you'll learn how to create the form and options JSONs. 

                
                    var optionsJSON = {};
                    var formJSON = {};
                    var Form1 = new iForm('#MyForm1', formJSON, optionsJSON);

Parameters

  • idSelector[HTML] (mandatory) - Space where form will be located
  • ArrayForm[JSON] (mandatory - Must to have at least 1 element) - Array JSON Object with the description of all field of form.
  • Options[JSON] (optional) - JSON Object with the options of form in general

Returns
iForm object with all its properties.



Options [JSON] (optional)

The JSON options parameter allows you configure the form element iForm wil create, it could includes the action, the encode method, and some texts for errors.

Example
                
                    var Options = {
                    formClass: 'form-group',
                    formMethod: 'POST',
                    enctype: 'multipart/form-data',
                    action: '#',
                    separator: '< br >',
                    textForSelect: 'Selecciona...',
                    textsErr: {
                    ErrMaxSize: 'El tamaño del archivo es mayor a _XX_ MB',
                    ErrFileAccept: 'El formato _XX_ es incorrecto o no es un archivo valido',
                    ErrMandatory: 'Este campo es obligatorio',
                    ErrUnique: 'La informacion  ya se encuentra  en otro campo',
                    ErrPatternAlph: 'Solo se permiten alphanumericos',
                    ErrPatternNumeric: 'Solo se permiten numericos',
                    ErrPattern: 'La informacion  no cincide con el patron',
                    ErrMinLength: 'El contenido por lo menos debe tener _XX_ caracteres de longitud',
                    ErrMaxLength: 'El contenido por lo mucho debe tener _XX_ caracteres de longitud',
                    ErrMinNum: 'El numero es menor a _XX_ ',
                    ErrMaxNum: 'El numero es mayor a  _XX_ ',
                    ErrIncludes: 'El contenido no incluye  _XX_ ',
                    ErrPass: 'El formato de contraseña es incorrecto ',
                    }
                    };


  • formClass (String)[form-group, userdefined] The class or classes you wanna stablish in your form (if youre working with gob.mx you mut to stablish 'form-group').
  • formMethod (string)[POST,PUT,GET,DELETE,UPDATE] The method you use for send the request across HTTP.
  • enctype (string)[multipart/form-data,application/x-www-form-urlencoded,text/plain] The encoding type you use for send the request across HTTP.
  • action (string)[userdefined] The endpoint for the HTTP request.
  • separator (string)[userdefined , HTML] The html content to separate fields one of each other.
  • textForSelect (string)[userdefined] Placeholder in select type field.
  • textsErr (JSON)[OBJECT] JSON node with user defined error texts for every validation.
    • ErrMaxSize (string)[userdefined] Text for error in validation max file sise. ('_XX_' will be replaced by number of size config by field )
    • ErrFileAccept (string)[userdefined] Text for error in validation file type. ('_XX_' will be replaced by number of size config by field )
    • ErrMandatory (string)[userdefined] Text for error in mandatory field.
    • ErrUnique (string)[userdefined] Text for error in repited field if unique.
    • ErrPatternAlph (string)[userdefined] Text for error only alphanumeric permited.
    • ErrPatternNumeric (string)[userdefined] Text for error only numeric permited.
    • ErrPattern (string)[userdefined] Text for error for pattern not match.
    • ErrMinLength (string)[userdefined] Text for error min length.('_XX_' will be replaced by number of length config by field )
    • ErrMaxLength (string)[userdefined] Text for error min length.('_XX_' will be replaced by number of length config by field )
    • ErrMinNum (string)[userdefined] Text for error min number.('_XX_' will be replaced by number of number config by field )
    • ErrMaxNum (string)[userdefined] Text for error min number.('_XX_' will be replaced by number of number config by field )
    • ErrIncludes (string)[userdefined] Text for error in content must to include.('_XX_' will be replaced by number of number config by field )
    • ErrPass (string)[userdefined] Text for error in password format incorrect.('_XX_' will be replaced by number of number config by field )

This is a JSON object that includes all the directions to build the fields and components for your form you can create the next type of fields automatically

Component:Text

To build a text form you can see the next example. 


                    {
                    id: 'req2',
                    label: 'Ingresa un texto ',
                    class: 'form-control',
                    type: 'text',
                    includes: 'thing',
                    mandatory: true,
                    unique: true,
                    name: 'req2',
                    style: '',
                    pattern: '',
                    minLength: 5,
                    maxLength: 100,
                    options:
                    {
                    validateOn: ['keyUp'],
                    skipUntilEndValidation: ['mandatory'],
                    forceLimits: true,
                    matchTo: 'id:req3',
                    matchToText: 'No coincide el campoa',
                    toUpperCase: true,
                    toLowerCase: false
                    }
                    }


  • id: [string](optional/Unique) The HTML id for the object in the DOM, This must be UNIQE.
  • label: [string](mandatory) The HTML label for this field.
  • class: [string](optional) The HTML class for this field.('form-control' must to be used if you use gob.mx framework)
  • placeholder: [number](optional) Text to show in field before you set content
  • type: [string](mandatory)[text,file,password,select,textarea,number] The type of field you want to build.
  • includes: [string](optional) If the text you imput must to include a part of tis value.
  • mandatory: [bolean](optional)[true/false] If the field is mandatory.
  • unique: [bolean](optional)[true/false] If the field is unique.
  • name: [string](optional) The HTML name for this field
  • style: [string](optional) The HTML style for this field
  • pattern: [string](optional) [number,alphanumeric,regexp] The HTML style for this field
  • minLength: [number](optional) The min length filed must be.
  • maxLength: [number](optional) The max length filed must be..
  • options:
    • validateOn: [array](optional)['keyUp','keyDown','Click','Change'] the event or events that trigger the validation.
    • skipUntilEndValidation: [array](optional)['mandatory','unique','pattern','minLength','maxLength','includes','matchTo'] Skips the validations that are executed in event triggers to call the metod validate
    • skipAtEndValidation: [array](optional)['mandatory','unique','pattern','minLength','maxLength','includes','matchTo'] Skips the validations at call the metod validate
    • forceLimits: [bolean](optional) Force some parameters to match as field needs before validation.
    • matchTo: [string](optional)['id:req'/'class:req'] Valididate the content of another field must be the same.
    • matchToText: [string](mandatory if match to exists) Text to show if no equal to match.
    • toUpperCase: [boelan](optional) Transform text or content in Uppercase format at keyup.
    • toLowerCase: [boelan](optional) Transform text or content in Lowercase format at keyup.
    • getDataAs: [string](optional) Returns the content to main object at change.

none

null

This element render the button for the form in the html space designed by you.
Params:

  • value: text for sumbit button
  • class: class for sumbit button
  • id: id for sumbit button
  • preventDefault: prevent default to form submit

Returns:
null


inicialize

null iForm.inicialize([optional](function));

This function inicialices all the functions for your form.
Params:

  • funtion: a callback function that runs at the end of validation inicialization

Returns:
null


getData

mixed iForm.getData([optional]([idex_element_number] | 'class:[classofelement] | 'id:[idofelemtn]' ,[optional](bolean));
